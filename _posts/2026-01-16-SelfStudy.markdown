# Chapter1. 데이터베이스 시스템

## 주요이론
1. 구조에 따른 데이터 분류
  - 정형 데이터: 행, 열 명확 EX엑셀
  - 반정형 데이터: 일정 구조 O, 고정 X, 키-값 형태 EX JSON, XML
  - 비정형 데이터: 구조 없다, EX 이미지, 영상, 텍스트 문서

2. 특성에 따른 데이터 분류
   - 범주형 데이터: 종류를 나타낸다. 질적 데이터
   - 수치형 데이터: 크기 비교와 산술적 연산이 가능, 양적데이터
  
3. 데이터베이스 시스템의 기본 구성 요소
  1. Database: 실제 데이터를 저장하는 공간
  2. DBMS(Database Management System): 데이터베이스를 관리하는 소프트웨어로 사용자와 데이터베이스 연결
  3. Data Model: 데이터를 설계하는 논리와 저장 기법
   +
  4. 데이터베이스 사용자
    - 데이터베이스를 이용하기 위해 접근하는 모든 사람
    - 일반사용자(최종사용자): 데이터를 조작(삽입·삭제·수정·검색)하는 사람, 주로 데이터 조작어를 이용
    - 개발자/ 관리자: DB설계, 운영, 보안관리
  
  
  5. 인터페이스 : 데이터베이스 언어(SQL),  응용프로그램

4. 데이터베이스 발전
   1. 파일시스템: 엑셀 화일과 전용 응용프로그램, 데이터 중복, 공유, 보안문제
   2. 데이터베이스 시스템: DB로 통합& 표준화, 데이터 중복 통제, 동시 공유, 장애발생 시 회복 가능, 응용프로그램 개발 비용 감소
   3. 웹 데이터베이스 시스템: 인터넷 연결
   4. 분산 데이터베이스 시스템

5. 파일 시스템과 DBMS (파일시스템 장점이 DBMS단점, 반대로 파일시스템 단점이 DBMS장점이 됨)
  (ex) 도서 검색프로그램
  if 파일시스템이 없다? : 도서 데이터를 프로그램에 저장해야한다. 즉, 새로운 책이 추가될 때마다 배열변수에 추가-재컴파일 해야한다
  1단계 도서데이터를 파일시스템(응용프로그램)에 저장
    - 데이터를 프로그램과 분리하여 별도의 파일에 저장& 파일 읽기 처리
    - 도서정보 구조변경 시 구조체 정보 수정 후 재작성
    - 같은 파일 공유시 문제 발생
  2단계 DBMS를 사용
    - 도서데이터 DB에 저장, DBMS로 관리
    - 데이터 독립성& 일관성& 중복 최소화
    - 프로그램이 간단하고 개발생산성이 높다
  
  DBMS 장단점
  
  장점
  - 데이터 중복 통제
  - 데이터 독립성 확보
  - 데이터 동시 공유 가능
  - 데이터 보안 향상
  - 데이터 무결성 유지
  - 표준화 가능
  - 장애 발생시 회복가능
  - 응용프로그램 개발 비용 준다

 단점
 - 도입비용이 많이 듬
 - 설계, 운영, 최적화, 튜닝에 전문지식 요구, 효율적 관리를 위해서는 숙련된 DBA(데이터베이스 관리자)가 필요하다

6. DBMS 발전과정
 - 1세대: 네트워크 DBMS(그래프형태), 계층 DBMS(트리 형태)
 - 2세대: 관계 DBMS(테이블 형태) (ex)오라클, 마리아
 - 3세대: 객체지향 DBMS, 객체관계 DBMS(객체지향+관계(ex)오라클)
 - 4세대: NoSQL DBMS(비정형 데이터를 처리), NewSQL DBMS: 관계 DBMS의 장점 + NoSQL의 확장성 및 유연성


7. 데이터베이스 언어 (SQL: Structured Query Language)
   - 사용자와 데이터베이스 관리 시스템 간의 통신 수단
   - 구성:데이터 정의어(DDL) : DBMS에 저장된 테이블 구조를 정의하는 데 사용, 데이터 조작어(DML) : 데이터를 검색·삽입·삭제·수정하는 데 사용/ 사용자가 쓴다, 데이터 제어어(DCL): 내부적으로 필요한 규칙이나 데이터의 사용 권한을 관리하는 데 사용
   - 핵심: 데이터 조작어 중 데이터를 검색하는 질의문(SELECT-FROM-WHERE 구조)


8. 데이터모델
 - 데이터를 DB 구조로 변환하는 논리적 가이드라인\
 - 계층/네트워크/관계/객체 모델등이 존재(관계 데이터 모델을 가장 많이 사용)

9. 데이터베이스의 개념적 구조
  - 3단계 데이터베이스 구조: 외부 단계, 개념 단계, 내부 단계
  - 스키마: 데이터베이스에 저장되는 데이터 구조와 제약조건을 정의
  - 인스턴스:스키마에 따라 데이터베이스에 실제로 저장된 값
  - 외부단계: 개별 사용자 관점, 여러 개의 외부 스키마가 있을 수 있음
  - 개념 단계 : 조직 전체의 관점, 통합 조직별로 하나만 존재하며 DBA가 관리함
  - 내부 단계 : 저장 장치의 관점, 하나의 데이터베이스에는 내부 스키마 하나만 존재
