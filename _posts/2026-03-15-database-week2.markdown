# Chapter 2. 관계 데이터 모델

## 1. 관계 데이터 모델의 개념

관계 데이터 모델은 개념적 구조를 논리적 구조로 표현하는 논리적 데이터 모델로, 데이터를 **릴레이션(Relation)**이라는 2차원 테이블 구조로 저장합니다.

### 주요 용어 정리

| 용어 | 설명 |
| --- | --- |
| **릴레이션(Relation)** | 행과 열로 구성된 테이블 |
| **속성(Attribute)** | 릴레이션의 열(Column), 파일 시스템의 필드(Field)에 해당 |
| **투플(Tuple)** | 릴레이션의 행(Row), 파일 시스템의 레코드(Record)에 해당 |
| **도메인(Domain)** | 속성 하나가 가질 수 있는 값들의 집합 |
| **차수(Degree)** | 속성의 전체 개수, 비교적 정적 |
| **카디널리티(Cardinality)** | 투플의 전체 수, 데이터 변화에 따라 자주 변하는 동적 성질 |
| **스키마(Schema)** | 릴레이션의 논리적 구조(내포)를 정의한 것 |
| **인스턴스(Instance)** | 스키마에 실제로 저장된 데이터의 집합(외연) |

### 릴레이션의 특징

| 특징 | 설명 |
| --- | --- |
| **속성의 원자성** | 모든 속성값은 더 이상 분해할 수 없는 단일값이어야 한다 |
| **속성의 무순서성** | 속성 간의 순서는 무의미하다 |
| **투플의 유일성** | 동일한 투플이 중복되어 존재할 수 없다 |
| **투플의 무순서성** | 투플 사이의 순서는 무의미하다 |

## 2. 무결성 제약조건 (Integrity Constraints)

데이터에 결함이 없는 정확하고 유효한 상태를 유지하기 위한 규칙입니다.

### 키(Key)의 종류

| 키 | 설명 |
| --- | --- |
| **슈퍼키(Super Key)** | 유일성을 만족하는 속성 또는 속성들의 집합 |
| **후보키(Candidate Key)** | 유일성과 **최소성**을 모두 만족하는 속성 집합 |
| **기본키(Primary Key)** | 후보키 중 선택된 메인 키, **NULL 값을 가질 수 없음** |
| **대체키(Alternate Key)** | 기본키로 선택되지 못한 나머지 후보키 |
| **외래키(Foreign Key)** | 다른 릴레이션의 기본키를 참조하여 릴레이션 간의 관계를 표현 |
| **대리키(Surrogate Key)** | 기본키가 복잡하거나 없을 때 인위적으로 만든 가상의 속성(일련번호 등) |

### 무결성 제약조건의 유형

| 제약조건 | 설명 |
| --- | --- |
| **개체 무결성 제약** | 기본키는 NULL일 수 없으며 중복되어서도 안 된다 |
| **참조 무결성 제약** | 외래키는 참조되는 릴레이션의 기본키 값과 일치하거나 NULL이어야 한다 |
| **도메인 무결성 제약** | 각 속성값은 정의된 도메인에 지정된 값만 가져야 한다 |

## 3. 관계대수 (Relational Algebra)

릴레이션에서 원하는 결과를 얻기 위한 절차를 기술하는 언어입니다.

### 일반 집합 연산자

> 두 릴레이션이 **합병 가능(차수와 도메인 일치)**해야 사용 가능한 연산이 있습니다.

| 연산자 | 기호 | 설명 |
| --- | --- | --- |
| **합집합** | ∪ | 두 릴레이션에 속한 모든 투플을 반환 |
| **교집합** | ∩ | 두 릴레이션에 공통으로 속한 투플을 반환 |
| **차집합** | − | 첫 번째 릴레이션에는 있지만 두 번째에는 없는 투플을 반환 |
| **카티션 프로덕트** | × | 두 릴레이션의 모든 투플을 각각 연결하여 가능한 모든 조합을 생성 |

### 순수 관계 연산자

| 연산자 | 기호 | 설명 |
| --- | --- | --- |
| **셀렉션** | σ | 조건에 맞는 투플(행)을 선택 |
| **프로젝션** | π | 필요한 속성(열)만 추출 |
| **조인** | ⋈ | 공통 속성을 기준으로 두 릴레이션을 결합 |
| **자연 조인** | | 동등 조인에서 중복된 조인 속성을 하나 제거한 형태 |
| **외부 조인** | (Left/Right/Full) | 조인 조건에 맞지 않는 데이터도 버리지 않고 NULL을 채워 반환 |
| **세미 조인** | | 자연 조인 후 한쪽 릴레이션의 결과만 반환하여 연산 비용을 줄임 |
| **디비전** | ÷ | 특정 조건을 **모두** 만족하는 대상을 찾을 때 사용 |
