# Chapter 1. 데이터베이스 시스템

## 주요이론

### 1. 구조에 따른 데이터 분류

| 분류 | 설명 | 예시 |
| --- | --- | --- |
| **정형 데이터** | 행과 열이 명확 | 엑셀 |
| **반정형 데이터** | 일정 구조는 있으나 고정되지 않음, 키-값 형태 | JSON, XML |
| **비정형 데이터** | 구조가 없음 | 이미지, 영상, 텍스트 문서 |

### 2. 특성에 따른 데이터 분류

| 분류 | 설명 |
| --- | --- |
| **범주형 데이터** | 종류를 나타내는 질적 데이터 |
| **수치형 데이터** | 크기 비교와 산술적 연산이 가능한 양적 데이터 |

### 3. 데이터베이스 시스템의 기본 구성 요소

1. **Database** — 실제 데이터를 저장하는 공간
2. **DBMS** (Database Management System) — 데이터베이스를 관리하는 소프트웨어로, 사용자와 데이터베이스를 연결
3. **Data Model** — 데이터를 설계하는 논리와 저장 기법
4. **데이터베이스 사용자** — 데이터베이스를 이용하기 위해 접근하는 모든 사람
   - **일반 사용자(최종 사용자):** 데이터를 조작(삽입·삭제·수정·검색)하는 사람, 주로 데이터 조작어를 이용
   - **개발자/관리자:** DB 설계, 운영, 보안 관리
5. **인터페이스** — 데이터베이스 언어(SQL), 응용프로그램

### 4. 데이터베이스 발전

1. **파일시스템** — 엑셀 파일과 전용 응용프로그램. 데이터 중복, 공유, 보안 문제 존재
2. **데이터베이스 시스템** — DB로 통합 및 표준화. 데이터 중복 통제, 동시 공유, 장애 발생 시 회복 가능, 응용프로그램 개발 비용 감소
3. **웹 데이터베이스 시스템** — 인터넷 연결
4. **분산 데이터베이스 시스템**

### 5. 파일 시스템과 DBMS

> 파일시스템의 장점이 DBMS의 단점, 반대로 파일시스템의 단점이 DBMS의 장점이 된다.

**예시: 도서 검색 프로그램**

- **파일시스템이 없다면?** — 도서 데이터를 프로그램에 직접 저장해야 하므로, 새로운 책이 추가될 때마다 배열변수에 추가 후 재컴파일 필요

**1단계: 파일시스템(응용프로그램)에 저장**
- 데이터를 프로그램과 분리하여 별도의 파일에 저장 및 파일 읽기 처리
- 도서정보 구조 변경 시 구조체 정보 수정 후 재작성 필요
- 같은 파일 공유 시 문제 발생

**2단계: DBMS 사용**
- 도서데이터를 DB에 저장, DBMS로 관리
- 데이터 독립성, 일관성, 중복 최소화
- 프로그램이 간단하고 개발 생산성이 높음

#### DBMS 장단점

**장점**
- 데이터 중복 통제
- 데이터 독립성 확보
- 데이터 동시 공유 가능
- 데이터 보안 향상
- 데이터 무결성 유지
- 표준화 가능
- 장애 발생 시 회복 가능
- 응용프로그램 개발 비용 절감

**단점**
- 도입 비용이 많이 듦
- 설계, 운영, 최적화, 튜닝에 전문지식 요구 — 효율적 관리를 위해서는 숙련된 DBA(데이터베이스 관리자)가 필요

### 6. DBMS 발전과정

| 세대 | DBMS 종류 | 데이터 구조 |
| --- | --- | --- |
| **1세대** | 네트워크 DBMS, 계층 DBMS | 그래프 형태, 트리 형태 |
| **2세대** | 관계 DBMS | 테이블 형태 (예: Oracle, MariaDB) |
| **3세대** | 객체지향 DBMS, 객체관계 DBMS | 객체지향 + 관계 (예: Oracle) |
| **4세대** | NoSQL DBMS, NewSQL DBMS | 비정형 데이터 처리 / 관계 DBMS 장점 + NoSQL 확장성 및 유연성 |

### 7. 데이터베이스 언어 (SQL: Structured Query Language)

사용자와 데이터베이스 관리 시스템 간의 **통신 수단**

| 구성 | 설명 |
| --- | --- |
| **DDL** (데이터 정의어) | DBMS에 저장된 테이블 구조를 정의하는 데 사용 |
| **DML** (데이터 조작어) | 데이터를 검색·삽입·삭제·수정하는 데 사용 (사용자가 주로 사용) |
| **DCL** (데이터 제어어) | 내부적으로 필요한 규칙이나 데이터의 사용 권한을 관리하는 데 사용 |

> **핵심:** 데이터 조작어 중 데이터를 검색하는 질의문 — `SELECT-FROM-WHERE` 구조

### 8. 데이터모델

- 데이터를 DB 구조로 변환하는 **논리적 가이드라인**
- 계층 / 네트워크 / 관계 / 객체 모델 등이 존재 (관계 데이터 모델을 가장 많이 사용)

### 9. 데이터베이스의 개념적 구조

**3단계 데이터베이스 구조**

| 단계 | 관점 | 설명 |
| --- | --- | --- |
| **외부 단계** | 개별 사용자 관점 | 여러 개의 외부 스키마가 있을 수 있음 |
| **개념 단계** | 조직 전체의 관점 | 조직별로 하나만 존재하며 DBA가 관리 |
| **내부 단계** | 저장 장치의 관점 | 하나의 데이터베이스에 내부 스키마 하나만 존재 |

- **스키마:** 데이터베이스에 저장되는 데이터 구조와 제약조건을 정의
- **인스턴스:** 스키마에 따라 데이터베이스에 실제로 저장된 값

### 2. DB 자격증 조사

#### 국가공인 데이터 자격증 (전문가 vs 일반)

국내에서 가장 인지도 높은 자격증으로, **한국데이터산업진흥원(Kdata)**에서 주관한다.

| 구분 | 전문가 (Professional) | 일반/준전문가 (Developer/Associate) | 특징 |
| --- | --- | --- | --- |
| **SQL** | **SQLP** (SQL 전문가) | **SQLD** (SQL 개발자) | SQL 활용 및 튜닝 능력 검증 (실무 핵심) |
| **데이터 설계** | **DAP** (데이터 아키텍처 전문가) | **DAsP** (데이터 아키텍처 준전문가) | 데이터 모델링 및 아키텍처 설계 역량 |
| **데이터 분석** | **ADP** (데이터 분석 전문가) | **ADsP** (데이터 분석 준전문가) | 통계 및 데이터 마이닝 분석 능력 |

> **참고:** 전문가 등급(SQLP, DAP, ADP)은 관련 학력이나 경력 등 **응시 자격 제한**이 있으나, 일반 등급은 제한 없이 누구나 응시 가능합니다.

#### DBMS 제조사별 국제 자격증

특정 소프트웨어 운영 및 관리 능력을 증명하는 자격증입니다.

**Oracle (오라클)** — 가장 대표적인 국제 DB 자격증으로, 단계별 취득이 일반적

| 단계 | 설명 |
| --- | --- |
| **OCA** (Associate) | 기초 단계 (운영 및 관리 기본) |
| **OCP** (Professional) | 실무자 단계 (**국내 취업 시장 선호도 1위**) |
| **OCM** (Master) | 최상위 전문가 단계 (실기 중심) |

**Microsoft (SQL Server/Azure)** — 최근 클라우드 데이터 서비스 중심으로 개편

| 자격증 | 설명 |
| --- | --- |
| **DP-900** | Microsoft Azure Data Fundamentals (기초) |
| **DP-300** | Administering Microsoft Azure SQL Solutions (전문가) |
